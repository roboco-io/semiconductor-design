# 세대 2 — 단일 시드 1등은 위양성이었고, 5-시드 median 평가 도구를 남겼습니다

> 한 후보가 단일 평가에서 1등으로 뽑혔으나, 평가를 5번 반복하니 셋 중 꼴찌였습니다. 평가용
> 데이터가 53행으로 적어 어떤 데이터가 뽑히느냐에 점수가 크게 출렁였기 때문입니다. 이 세대는
> 새 baseline을 만들지 못했지만, 이후 모든 세대가 쓰게 될 **5-시드 median 평가 도구**를 남겼습니다.

## 실험 전제

세대 1에서 에이전트(`train.py`를 자동으로 변형하는 LLM)가 baseline(현재 기준 모델, 직전까지의
최고 모델)을 이기는 후보를 내놓았습니다. 세대 2의 질문은 두 가지입니다.

1. 다시 시도해도 또 baseline을 이기는 후보가 나오는가.
2. 이긴 결과가 진짜 개선인가, 아니면 평가 과정의 우연(운)인가.

두 번째 질문이 특히 중요합니다. 우연히 한 번 좋게 나온 후보를 진짜 개선으로 착각하면, 실제로는
열등한 모델을 새 baseline으로 승격시켜 이후 모든 세대의 기준을 오염시키게 됩니다. 이 세대는
바로 그 위험을 실제로 마주쳤습니다.

## 방법

surrogate 모델(칩 설계를 끝까지 진행하지 않고도 최종 타이밍 성능을 예측하는 회귀 모델)을 학습하는
`train.py`를 두 에이전트가 각자 변형해 후보 2개를 만들었습니다.

- **cand-000 (claude, conservative)** — 모델이 참고하는 feature(예측에 쓰는 입력 정보 조각)를
  13개 추가했습니다(예: 단계별 지연의 합, 최댓값과 평균의 차이). 모델은 서로 다른 두 회귀기
  (HistGradientBoosting + ExtraTrees)의 예측을 6:4 가중으로 평균하는 앙상블입니다.
- **cand-001 (codex, moderate)** — feature를 37개 추가했고(로그·제곱·곱 조합 포함), 회귀기도
  네 개(HistGradientBoosting + ExtraTrees + RandomForest + GradientBoosting)를 가중 평균하는
  더 큰 앙상블을 구성했습니다. feature와 모델을 모두 늘려 예측력을 높이려는 전략입니다.

평가 방식에 함정이 있었습니다. 세대 2는 **단일 시드(seed=0)**로 한 번만 평가했습니다. 여기서
시드(seed)란 가진 데이터를 학습용(train)과 검증용(val)으로 나눌 때 그 분할을 결정하는 난수입니다.
시드를 바꾸면 어떤 행이 검증용으로 뽑히는지가 달라집니다.

검증 데이터가 **53행**밖에 없어서, 어떤 53행이 검증용으로 뽑히느냐에 따라 점수가 크게 달라집니다.
한 번의 분할만 보면 그 점수가 후보의 실력인지 분할 운인지 구분할 수 없습니다. 이것이 단일 시드
평가의 위험입니다.

## 결과

### 단일 시드(seed=0) 평가 — cand-001이 1등으로 보임

지표는 `val_mae`(검증 데이터에서의 평균 예측 오차, 낮을수록 정확)입니다.

| 후보 | val_mae (seed=0) | 단일 시드 판정 |
|---|---|---|
| cand-001 (codex, moderate) | 0.0992 | 최저 → winner로 선정 |
| cand-000 (claude, conservative) | 0.1022 | |
| baseline (gen-001 winner) | 0.1042 | |

단일 시드만 보면 cand-001이 baseline보다 낮아 새 baseline 후보로 보였습니다.

### 5-시드 재평가(seed 0~4) — 점수가 약 3배 출렁였고 순위가 뒤집힘

같은 후보를 검증 분할만 바꿔 다섯 번 평가하고, 그 중앙값(median)으로 순위를 다시 매겼습니다.
median은 다섯 점수를 크기순으로 줄 세웠을 때 한가운데 값으로, 분할 운에 의한 변동을 덜 탑니다.

| 후보 | per-seed val_mae [0,1,2,3,4] | median |
|---|---|---|
| baseline (gen-001 winner) | 0.1042, 0.1557, 0.0831, 0.0865, 0.0696 | **0.0865 (최저)** |
| cand-000 (claude) | 0.1022, 0.1602, 0.0891, 0.0786, 0.0567 | 0.0891 |
| cand-001 (codex) | 0.0992, 0.1571, 0.0753, 0.1153, 0.0562 | 0.0992 (최악) |

두 가지가 드러났습니다.

1. **분할 운이 점수를 크게 흔든다.** 같은 후보인데 시드에 따라 약 0.056에서 0.16까지, 약 3배
   차이가 났습니다. 검증 데이터가 53행으로 적기 때문입니다.
2. **단일 시드 1등이 median에서는 꼴찌가 됐다.** seed=0이 우연히 cand-001에 유리한 분할이었을
   뿐입니다. baseline의 median(0.0865)을 이긴 후보는 없었습니다.

### 산출물

새 baseline은 나오지 않았지만, 이후 세대가 계속 쓰게 될 두 가지를 남겼습니다.

- **`rejudge.md`** — 5-시드 median 재판정 기록. cand-001은 단일 시드 위양성으로 판정되어
  reject(미승격, baseline 유지)되었고, gen-001 winner가 baseline으로 유지됨을 명시합니다.
- **5-시드 median selection harness** — 이후 평가의 기본 도구입니다. 모든 후보를 seed 0~4로
  다섯 번 학습·평가하고 median으로만 순위를 정합니다. 규칙도 정했습니다. 다섯 중 한 번이라도
  계산이 발산(inf)하면 즉시 탈락시키고, 다섯 점수를 모두 기록해 사후에 운의 영향을 검증할 수
  있게 했습니다. 이 도구는 세대 3 이후 모든 세대의 평가 기준이 되었습니다.

## 분석

핵심은 **평가 프로토콜을 단일 시드에서 5-시드 median으로 바꾼 것만으로 순위가 뒤집혔다**는
점입니다(근거: `rejudge.md`, `generation.json`). 즉 세대 1의 "에이전트가 baseline을 이겼다"는
결과 일부는 측정 아티팩트였을 가능성이 있습니다.

이 세대는 본 프로젝트의 검증 가설(H-B: 검증 게이트가 위양성 승격을 차단한다)을 실제로 입증한
사례입니다. 5-시드 재평가가 없었다면 분할 운으로 한 번 좋게 나온 cand-001이 새 baseline으로
승격되어 이후 기준을 오염시켰을 것입니다. 미승격은 실패가 아니라, 오염을 막고 더 신뢰할 수 있는
평가 도구를 확보한 결과입니다.

## 용어

- **seed(시드)** — 데이터를 학습용/검증용으로 나누는 분할을 결정하는 난수. 바꾸면 검증에 쓰이는
  행이 달라진다.
- **단일 시드(single seed)** — 시드 하나로 한 번만 평가하는 것. 데이터가 적으면 분할 운에 크게
  휘둘려 위험하다.
- **median(5-seed)** — 시드 0~4로 다섯 번 평가한 val_mae의 중앙값. 분할 운에 의한 변동을 억제한다.
- **val_mae** — 검증 데이터에서의 평균 예측 오차(MAE). 낮을수록 정확.
- **feature** — 모델이 예측에 참고하는 입력 정보 조각. 많다고 항상 정확해지는 것은 아니다.
- **위양성(false positive)** — 실제로는 우수하지 않은데 우연히 좋아 보이는 결과.
- **baseline** — 현재 기준 모델. 후보는 이를 median에서 이겨야 승격된다.
