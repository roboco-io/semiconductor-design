# 세대 1 — 에이전트가 사람이 만든 기준 모델을 처음으로 능가하다

> 이 프로젝트의 첫 세대입니다. 진짜 칩 하나를 끝까지 설계해 53행짜리 데이터셋을 확보했고,
> 두 LLM 에이전트가 학습 스크립트를 자동으로 변형했습니다. 그중 한 후보가 사람이 작성한
> baseline(현재 기준 모델)을 넘어서 새 기준으로 승격되었으며, 그 과정에서 실제 코드 결함도 하나 발견했습니다.

## 🎯 실험 전제

이 프로젝트는 surrogate 모델(칩 설계를 끝까지 진행하지 않고도 최종 타이밍 성능을 예측하는 회귀 모델)을
만듭니다. 칩을 완성하는 EDA flow(설계 자동화 공정)는 시간이 오래 걸리므로, 공정 중간에 나오는 정보만으로
최종 결과를 미리 예측할 수 있으면 비용을 크게 줄일 수 있습니다.

첫 세대가 던진 질문은 다음과 같습니다. **이 surrogate를 학습시키는 일에서, LLM 에이전트가 사람이
손으로 작성한 baseline보다 더 나은 방법을 스스로 찾아낼 수 있는가?**

## 🧪 방법

### 데이터셋 준비

먼저 `gcd`라는 실제 칩 설계 하나를 EDA flow로 끝까지 실행했습니다. 이 작업은 AWS Fargate에서
native x86(에뮬레이션 없는 x86 환경)으로 1회 수행했고, 공정 중 생성된 두 보고서(합성 보고서 `synth.rpt`,
배선 보고서 `route.rpt`)를 수집했습니다. 보고서를 정리하니 칩 내부의 신호 경로 53개가 행으로 나왔고,
이것이 53행짜리 학습 데이터셋입니다. 각 행은 8개의 입력 feature(예측에 쓰는 입력 특성)와
정답 레이블(경로의 최종 슬랙)로 구성됩니다.

### 두 에이전트의 변형 전략

surrogate를 학습시키는 스크립트는 `train.py`(에이전트가 변형하는 유일한 파일)입니다. 두 에이전트
(claude, codex)에게 이 스크립트를 자유롭게 변형하도록 맡겼습니다.

| 후보 | 에이전트 | 전략 | 변형 내용 |
|---|---|---|---|
| **cand-000** | claude | 보수적(conservative) | 모델 종류는 유지하고, HistGradientBoosting(잔차를 단계적으로 줄이는 부스팅 회귀)의 손실 함수를 평가 지표와 일치시키고 early stopping(과적합 전 학습 중단)을 소폭 조정 |
| **cand-001** | codex | 중간 규모(moderate) | VotingRegressor(여러 모델의 예측을 평균내 결합하는 앙상블)로 HGB와 ExtraTrees를 묶고, 파생 feature(기존 입력을 조합해 만든 새 입력 특성)를 추가 |

cand-001의 핵심은 파생 feature 설계입니다. 원래의 8개 입력에 더해, 경로가 얼마나 빠듯한지를 나타내는
criticality, 단계당 여유를 나타내는 slack_per_stage 등 13개의 파생 feature를 추가했습니다. 모델이 더
쉽게 학습할 수 있도록 입력 표현을 가공한 것입니다.

### 평가 방식

변형한 스크립트로 각 후보를 학습시키고, val_mae(검증 데이터에서의 평균 예측 오차, 낮을수록 정확)를
측정했습니다. 학습에 쓰지 않은 부분으로 평가하는 설계-분리(holdout) 방식을 적용해, 단순 암기가 아닌
실제 일반화 성능을 확인했습니다.

## 📊 결과

| 모델 | 작성 주체 | val_mae (낮을수록 정확) |
|---|---|---|
| naive(상수 예측 기준선) | — | **1.41** |
| baseline | 사람 | **0.18** |
| cand-000 | claude | **0.129** |
| **cand-001 (winner)** | **codex** | **0.104** |

두 후보 모두 사람이 작성한 baseline(0.18)을 능가했습니다. 그중 cand-001이 val_mae 0.104로 가장 낮아,
naive 기준선(1.41)보다 약 13배 정확했습니다. cand-001은 모든 검증을 통과해 새 baseline으로
승격(promoted)되었습니다.

### 산출물

- **`candidates/cand-001/art/model.joblib`** — codex가 학습시킨 surrogate 모델을 직렬화한 파일.
  이 파일을 불러오면 실제로 칩 성능을 예측할 수 있습니다. cand-000의 모델 파일도 함께 저장되어 있습니다.
- **`generation.json`** — 이 세대의 결과 요약. winner(`cand-001`), 점수(`0.104`),
  상태(`promoted`)가 기록되어 있습니다.
- **`results.tsv`** — 두 후보의 점수를 나란히 정리한 채점 결과.
- **git tag `gen-001-best`** — 승격된 버전에 붙인 태그로, 나중에 해당 세대의 winner를 바로 찾게 합니다.
  이 winner 스크립트가 메인 `train.py`가 되어 다음 세대의 baseline이 됩니다.
- **`revalidation.md`** — 이후(2026-06-08) 결과가 우연이 아닌지 다시 엄격하게 검증한 기록입니다.
  T1 게이트가 50-fold 재심을 수행했고, winner가 baseline과 통계적으로 구별 가능(distinguishable)함을
  확인했습니다.

## 💡 분석

이 세대는 두 가지 가설을 뒷받침합니다.

- **H-A — 에이전트가 사람을 능가할 수 있다.** codex는 단순히 하이퍼파라미터만 바꾼 것이 아니라,
  앙상블 결합과 파생 feature 설계라는 방법론적 개선을 스스로 도입해 baseline을 넘어섰습니다.
- **H-B — 승격은 검증을 거친다.** 후보가 점수가 낮다고 해서 곧바로 baseline이 되지는 않습니다.
  게이트를 통과해야만 승격되며, 이는 우연에 의한 가짜 우승을 걸러냅니다.

또한 실험 과정에서 실제 코드 결함이 드러났습니다. cand-001은 직접 만든 파생 feature 변환 함수를
FunctionTransformer로 모델에 포함시켰는데, 이 함수가 `__main__`을 참조한 상태로 pickle되어
holdout 재채점 시 모델을 다시 불러올 수 없었습니다. 이는 `holdout.py`를 보강해 해결했습니다.
실제로 학습을 돌려봐야만 드러나는 종류의 마찰입니다.

## 💬 용어

- **surrogate 모델**: 칩 설계를 끝까지 진행하지 않고도 최종 타이밍 성능을 예측하는 회귀 모델.
- **`train.py`**: surrogate를 학습시키는 스크립트. 에이전트가 변형하는 유일한 파일.
- **val_mae**: 검증 데이터에서의 평균 예측 오차(MAE). 낮을수록 정확.
- **baseline**: 현재 기준 모델. 직전까지의 최고 모델이며, 후보는 이를 이겨야 승격된다.
- **VotingRegressor**: 여러 회귀 모델의 예측을 평균내 결합하는 앙상블 기법.
- **파생 feature(feature engineering)**: 기존 입력을 조합해 모델이 학습하기 쉬운 새 입력 특성을 만드는 것.
- **승격(promote)**: 후보가 검증을 통과해 새 baseline이 되는 것.
- **T1 게이트**: 평가를 여러 번 반복해 baseline 대비 개선이 통계적으로 유의한지 검정하는 재심 게이트.
