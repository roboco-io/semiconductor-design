# 세대 7 이야기 — "누가 이겼나"와 "확실히 이겼나"는 다른 질문 🤔

> 세대 6에서 **고친 심판**으로 처음 시험을 봤어요. 1등 도전자가 깜짝 시험(LODO)은 통과했는데,
> 심판(T1)이 "그건 운일 수도 있어, 확실하지 않아"라며 챔피언 자리를 안 내줬어요.
> 알고 보니 **두 심판이 서로 다른 질문**을 던지고 있었던 거예요.

이 이야기를 읽기 전에 알아두면 좋은 등장인물(비유)이에요. 세대마다 똑같이 쓰니까 한 번만 익혀두면 돼요.

| 진짜 이름 | 이 이야기에서 부르는 말 |
|---|---|
| surrogate 모델 | 칩이 완성되기 전에 성능을 미리 맞히는 **점쟁이 기계** |
| `train.py` | 점쟁이를 가르치는 **레시피** |
| 에이전트(claude/codex) | 레시피를 맘대로 바꿔보는 **로봇 요리사** |
| `val_mae` | 예측이 실제와 얼마나 **빗나갔나** (작을수록 똑똑함) |
| baseline | 지금 1등 점쟁이 = **챔피언** |
| median(5 seed) | 운인지 보려고 **5번 시험봐 가운데 점수**로 판단 |
| LODO | 한 번도 안 가르친 칩으로 보는 **깜짝 시험** |
| 교차설계 T1 | 그게 운이 아니라 **확실한지** 통계로 따지는 **심판** |
| Codex 게이트 | 꼼수 없나 보는 **다른 회사 검사관** |
| 설계(group) | 성격이 서로 다른 **칩 종류** (gcd·aes·ibex·jpeg) |

## 🎯 알고 싶었던 것

세대 6에서 우리는 심판이 **엉뚱한 시험지**를 채점하던 걸 발견하고 심판을 고쳤어요
(교차설계 T1 — "처음 보는 칩"으로만 채점하게 바꿨죠).

그래서 세대 7의 질문은 딱 하나예요:

> **고친 심판으로 다시 돌리면, 드디어 새 챔피언이 나올까?**

이번에도 칩 종류는 **3가지**(aes·gcd·ibex)를 섞어 썼어요. 이 셋은 성격이 꽤 다른 칩이에요 —
특히 `ibex`는 숫자가 다른 칩들보다 **자릿수 단위로 큰**, 좀 별난 칩이라 늘 말썽을 일으켜요.

## 🧪 우리가 한 일

로봇 요리사 4명에게 레시피를 마음대로 바꾸게 하고, 4단 게이트로 채점했어요.

**후보들이 시도한 방법** (12살 번역):

| 후보 | 요리사 | 성격 | 무슨 꾀를 냈나 (쉬운 말로) |
|---|---|---|---|
| **cand-002** ⭐ | claude | 과감 | 칩마다 숫자 단위가 달라서 헷갈리니까, **숫자를 "비율"로 바꿔** 단위 차이를 지웠어요. 그리고 "합성 때 잰 여유값"을 기준점 삼아 거기서 **얼마나 달라지나**만 맞히게 했죠 (절대값 대신 변화량). |
| cand-001 | codex | 중간 | 점쟁이 여러 명(빠른 점쟁이 + 신중한 점쟁이)을 **투표**로 합쳤어요. 여럿이 모이면 한 명이 헛다리 짚어도 평균이 안정되니까요. |
| cand-003 | codex | 신중 | 챔피언 레시피를 거의 그대로 두고 **아주 조금만** 손봤어요. 모험보다 안전을 택한 거죠. |
| cand-000 | claude | 신중 | 깜짝 시험에서 자꾸 미끄러지길래 **규칙을 살짝 더 엄격하게**(과적합 방지) 조였어요. |

채점은 4단계로 했어요:
1. **median(5번 시험 가운데 점수)** — 운으로 한 번 잘 본 걸 1등으로 뽑지 않으려고.
2. **깜짝 시험(LODO)** — 한 번도 안 가르친 칩으로 "누가 더 많이 이기나" 세기.
3. **교차설계 T1 심판** — 그 차이가 "운이 아니라 확실한지" 통계로 따지기.
4. **Codex 검사관** — 꼼수(metric gaming) 없나 다른 회사가 검사. (T1을 통과해야만 부르는 마지막 관문)

## 📊 결과 (쉬운 말로)

### 1) 1등 도전자

`cand-002`(과감 요리사)가 빗나감 **1.29**로 1등! 이건 **지금까지 중 가장 낮은(똑똑한) 점수**예요.
"비율로 바꾸고 변화량만 맞히는" 꾀가 잘 먹힌 거죠.

| 후보 | median 빗나감 (낮을수록 똑똑) |
|---|---|
| cand-002 ⭐ | **1.29** |
| cand-001 | 2.40 |
| cand-003 | 3.80 |
| cand-000 | 3.94 |

### 2) 깜짝 시험(LODO) — **통과** ✅ (2/3 승)

처음 보는 칩 3종류로 도전자(cand-002)와 챔피언을 붙여봤어요. 숫자가 **낮을수록** 똑똑해요.

| 처음 본 칩 | 그냥 찍기(naive) | 챔피언(baseline) | 도전자(winner) | 누가 이겼나? |
|---|---|---|---|---|
| aes | 1.72 | 2.68 | **1.27** | 도전자 승 🟢 |
| gcd | 1.41 | 2.52 | **2.04** | 도전자 승 🟢 |
| ibex | 12.81 | 2.96 | 6.93 | **챔피언 승** 🔴 |

도전자가 **3종류 중 2종류에서 이겼으니까** 깜짝 시험은 "통과"(generalizes_better)예요.
"누가 더 많이 이겼나?"만 보면 도전자가 이긴 거죠.

### 3) 교차설계 T1 심판 — **"비겼다"** ❌ (탈락: rejected_t1)

그런데 심판은 다른 걸 물었어요: **"그게 진짜 확실한 승리야, 아니면 그냥 운이야?"**

심판은 처음 보는 칩들을 **30번이나 반복해서** 시험봤어요(3칩 × 10번). 그리고 평균 빗나감을 쟀죠:

| 모델 | 평균 빗나감 (30번 시험) |
|---|---|
| 그냥 찍기(naive) | 5.31 |
| 챔피언(baseline) | **3.80** |
| 도전자(winner) | 4.16 |

이런! 평균으로 보니 **도전자(4.16)가 챔피언(3.80)보다 오히려 조금 나빴어요.**
차이는 +0.36인데, 심판의 통계 자(自)로 재보니:

- 확실함 점수(p값) = **0.655** — 1에 가까울수록 "구별 안 됨". 0.655는 **전혀 확실하지 않다**는 뜻.
- 흔들림 범위(CI) = [-0.82 ~ +1.55] — **0을 가운데 두고 양쪽으로 걸쳐** 있어요. "더 좋다"고도 "더 나쁘다"고도 말 못 해요.

그래서 심판의 판정은 **indistinguishable(비김)** — "도전자와 챔피언을 구별할 수 없다." 챔피언 자리 유지!

> 💥 **왜 이렇게 됐을까? 범인은 `ibex`예요.** 도전자는 aes·gcd에서 깔끔하게 이겼지만,
> 별난 칩 ibex에서 **크게 졌어요**(6.93 vs 2.96). 이 큰 패배 하나가 평균을 끌어내려서,
> 다른 곳에서 번 이득이 **"노이즈(잡음)에 묻혀" 사라진** 거예요. 깜짝 시험은 "2번 이김"으로
> 봐주지만, 심판은 ibex의 큰 패배까지 다 합산해서 "확실한 개선 아님"이라고 본 거죠.

### 4) Codex 검사관 — 안 불렀어요

T1 심판을 통과 못 했으니, 마지막 관문인 Codex 검사관은 부르지도 않았어요(approve: False, "T1 미통과 — 심사 생략").

## 💡 이 실험이 가르쳐준 것

**"누가 더 많이 이겼나(LODO)"와 "확실히 이겼나(T1)"는 서로 다른 질문이에요!**

- **깜짝 시험(LODO)** 은 *방향*을 봐요 — "몇 종류 칩에서 이겼나?" (2 > 1이니까 통과).
- **T1 심판**은 *확실함*을 봐요 — "그 차이가 운이 아니라고 통계가 보증하나?" (한 칩에서 크게 지면 ❌).

두 심판이 **서로 다른 걸 보면서 팀으로** 일하니까, **가짜 우승자**를 정확히 걸러낼 수 있어요.
만약 깜짝 시험만 믿었다면 도전자가 챔피언이 됐을 거고, 그러면 ibex에서 크게 지는 약한 점쟁이로
챔피언이 바뀌어(=baseline이 오염돼) 다음 세대가 다 망가졌을 거예요.

또 하나 중요한 교훈: **점수가 낮다(median 1.29)고 일반화가 좋은 게 아니에요.** 도전자는 역대 최저
점수였는데도 심판은 "비김"이라 했어요. 거꾸로 세대 6 도전자는 점수가 더 높았는데도(3.50)
심판이 "확실히 이김"이라 했죠. **시험장(in-loop)에서 잘 본다 ≠ 처음 보는 칩에서도 잘한다.**

### 📄 결과물이 말해주는 것 (report.md "교차설계 T1" 섹션)

`report.md`의 3) 교차설계 T1 표가 바로 이 이야기의 증거예요. winner 평균(4.16)이 baseline(3.80)과
**거의 같고**, p=0.655라 "비김(indistinguishable)"이라고 적혀 있어요. 이 한 줄이 **"게이트가 위양성
(가짜 우승)을 정상적으로 막았다"** 는 뜻이에요 — 탈락은 실패가 아니라, 심판들이 제대로 일했다는 증거죠.

## 💬 어려운 말 풀이

- **교차설계 T1**: 세대 6에서 고친 심판. 깜짝 시험처럼 "처음 보는 칩"으로 재되, **그 차이가 확실한지**를 통계(p값·흔들림 범위)로 따져요.
- **indistinguishable(비김)**: 도전자와 챔피언이 통계적으로 구별이 안 됨. "확실한 승부가 안 났다"는 뜻이에요.
- **p값(확실함 점수)**: 0에 가까우면 "확실히 다르다", 1에 가까우면 "구별 안 된다". 세대 7은 0.655라 구별 불가.
- **CI(흔들림 범위)**: 진짜 차이가 들어있을 법한 구간. 이 구간이 0을 걸치면 "더 좋다"고 단정 못 해요.
- **ibex**: 다른 칩들과 숫자 단위가 자릿수로 다른 별난 칩. 네 세대 내내 일반화의 병목(말썽꾼)이었어요.
