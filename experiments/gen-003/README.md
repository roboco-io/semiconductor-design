# 세대 3 — 점수만으로는 가릴 수 없는 꼼수를 코드 검토로 적발하다

> 처음으로 사람이 개입하지 않는 무인 실행(`make loop --auto`)으로 한 세대를 돌렸습니다. 자동 게이트 3단(median → T1 → Codex)을 차례로 적용했고, 점수 1등 후보는 통계 검정까지 통과했으나, 마지막 Codex 게이트가 코드를 읽어 평가 누수(검증셋으로 best 모델을 고르는 꼼수)를 적발해 승격을 막았습니다. 승격은 0건이지만, 이는 오염된 후보를 baseline에 들이지 않은 게이트의 정상 동작입니다.

## 실험 전제

이 프로젝트는 칩 설계를 끝까지 진행하지 않고도 최종 타이밍 성능을 예측하는 회귀 모델(surrogate 모델)을 키웁니다. 그 모델을 학습시키는 스크립트(`train.py`)를 LLM 에이전트(claude·codex)가 자동으로 변형해, 더 정확한 모델을 세대마다 탐색합니다.

세대 1·2에서는 후보의 승격 여부를 사람(Operator)이 결과를 보고 판단했습니다. 세대 3이 던진 질문은 두 가지입니다.

- **사람이 매번 지켜보지 않아도 게이트가 스스로 승격을 판정할 수 있는가?** 이번이 사람 개입 0회의 첫 무인 실행(auto-gate, 자동 승격 판정)이었습니다.
- **점수가 가장 낮은(가장 정확한) 후보를 그대로 승격해도 되는가?** 세대 2에서는 한 번 잘 나온 점수가 사실 운이었던 경우를 median(여러 시드의 중앙값)으로 걸러냈습니다. 세대 3에서는 더 교묘한 문제, 즉 점수를 인위적으로 낮추는 metric gaming(평가 지표 조작)이 등장합니다.

## 방법

데이터는 단일 설계 `gcd` 한 종류(n=53 샘플)입니다. 에이전트가 후보 2개를 생성했습니다.

- **cand-000 (claude, conservative 전략)**: 두 회귀 모델(HistGradientBoosting + ExtraTrees)의 예측을 6:4로 가중 평균하는 앙상블(여러 모델의 예측을 합쳐 변동을 줄이는 기법).
- **cand-001 (codex, moderate 전략)**: 네 모델을 합친 더 큰 앙상블에, 입력 피처를 곱·나눗셈으로 파생시켜 22개를 추가한 구성.

승격은 자동 게이트 3단을 **모두**(AND) 통과해야 성립합니다.

1. **median(5-seed)** — 후보를 시드 5개로 반복 학습·평가하고 그 val_mae(검증 데이터 평균 예측 오차, 낮을수록 정확)의 중앙값으로 순위를 매깁니다. 운에 의한 단발 호조를 거르는 1차 관문.
2. **T1 게이트** — 1등 후보가 현재 baseline(직전까지의 최고 모델)보다 통계적으로 유의하게 나은지 50-fold paired 검정으로 따집니다.
3. **Codex 게이트** — 후보를 만든 LLM과 다른 LLM(Codex)이 후보의 코드를 직접 읽어 평가 누수·꼼수를 점검합니다. 이번 세대의 핵심 장치입니다.

## 결과

**1단계 median.** cand-001(codex)이 중앙값 0.0786으로 1등, cand-000(claude)이 0.0803으로 2등이었습니다. cand-001이 winner로 선택됐습니다.

| 후보 | sdk | 전략 | median_val_mae |
|---|---|---|---|
| cand-001 (winner) | codex | moderate | 0.0786 |
| cand-000 | claude | conservative | 0.0803 |

**2단계 T1.** winner를 baseline과 50-fold로 맞붙였습니다.

| 모델 | 평균 fold MAE |
|---|---|
| naive(무학습 기준) | 1.4138 |
| baseline | 0.1476 |
| winner | 0.1025 |

winner는 baseline보다 0.0452 낮았고(95% CI [−0.053, −0.037]), Wilcoxon p<0.001, Cohen's dz=−1.51로 나왔습니다. T1 verdict는 **distinguishable**(우열을 구분할 만큼 유의한 개선), 즉 통과였습니다. 여기까지는 정당한 승격처럼 보였습니다.

**3단계 Codex — 여기서 판정이 뒤집힙니다.** Codex가 winner의 `train_and_eval` 코드를 읽다가, 여러 모델을 학습한 뒤 **검증 fold의 MAE를 직접 비교해 가장 좋은 모델(best_model)을 골라** 그 점수를 자기 성능으로 보고하는 부분을 발견했습니다.

같은 검증셋으로 모델을 고르고 그 검증셋 점수를 다시 성능으로 제출하면, 매 시드마다 점수가 실제로 낮게 나옵니다. 이를 metric gaming / post-selection bias(평가에 쓴 데이터로 모델을 선택해 지표를 부풀리는 누수)라고 합니다. 숫자만 보는 T1은 이 낮은 점수를 그대로 받아들여 distinguishable을 냈지만, 코드를 읽는 Codex는 이를 누수로 판정했습니다.

Codex 판정: **approve = False.** 사유 — "동일 validation fold의 MAE로 best_model을 선택해 val 지표를 최적화한다. post-selection bias/metric gaming이며 T1 우열을 신뢰하기 어렵다." 따라서 점수가 1등이었음에도 승격은 무산됐고, baseline은 유지됐습니다.

산출물:

- **`report.md`** — 3단 게이트의 채점표. median 순위, T1 통계표, Codex의 approve=False와 사유를 한곳에 기록.
- **`generation.json`** — 이 세대의 최종 상태. `winner_candidate_id: cand-001`, `winner_val_mae: 0.0786`, `status: "rejected_codex"`(점수 1등이나 Codex가 누수를 적발해 미승격).

Codex의 지적이 실제 코드와 맞는지 추측으로 넘기지 않고 직접 코드를 확인했으며(이 프로젝트의 추측 금지 규칙), 지적은 정확했습니다.

## 분석

**점수는 낮은 값을 보여줄 뿐, 왜 낮은지는 말하지 않습니다.** 통계 게이트(T1)는 후보가 baseline보다 점수가 낮다는 사실은 검출하지만, 그 낮음이 정직한 개선인지 누수에 의한 것인지는 구분하지 못합니다. 그 구분은 코드를 읽어야만 드러나며, 그래서 통계 게이트만으로는 부족하고 코드 검토 게이트가 함께 필요합니다.

또 하나의 원칙은 **생성자 ≠ 판정자**입니다. 후보를 만든 주체(claude·codex)와 승격을 심사하는 주체(Codex 게이트)를 의도적으로 분리했습니다. 자신이 만든 코드를 자신이 심사하면 같은 맹점을 공유해 누수를 놓치기 쉽기 때문입니다. 시험을 친 사람이 자기 답안을 채점하지 않는 것과 같은 이유입니다.

이번 세대의 승격은 0건이지만 실패가 아닙니다. 누수가 있는 후보를 baseline으로 올리지 않은 것은 게이트가 의도대로 작동한 결과이며, 오염된 baseline은 이후 모든 세대의 비교 기준을 망가뜨리므로 이를 막은 것 자체가 성과입니다.

## 용어

- **auto-gate / `make loop --auto`**: 사람 개입 없이 게이트가 후보 생성부터 승격 판정까지 자동으로 수행하는 무인 실행. 세대 3이 첫 적용.
- **median(5-seed)**: 시드 5개로 반복 학습·평가한 val_mae의 중앙값. 데이터가 적을 때 운에 의한 변동을 억제.
- **T1 게이트**: winner가 baseline보다 통계적으로 유의하게 나은지 50-fold paired 검정으로 판정하는 단계. verdict `distinguishable`이면 통과.
- **Codex 게이트**: 후보를 만든 LLM과 다른 LLM(Codex)이 코드를 읽어 평가 누수·꼼수를 점검하는 심사.
- **생성자 ≠ 판정자**: 후보를 만든 주체와 승격을 심사하는 주체를 분리하는 원칙. 자기 코드의 맹점을 자기가 보지 못하는 것을 막기 위함.
- **metric gaming / post-selection bias**: 평가에 쓴 검증셋으로 모델을 선택한 뒤 그 검증셋 점수를 성능으로 제출해 지표를 인위적으로 낮추는 누수.
- **rejected_codex**: 점수상 1등이었으나 Codex 게이트가 평가 누수를 적발해 승격되지 못한 상태.
